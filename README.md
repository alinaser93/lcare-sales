# LCare — تطبيق إدارة مبيعات مستشعرات السكر (CGM)

تطبيق ويب متكامل لإدارة مبيعات مستشعرات السكر، يعمل بالكامل في المتصفح ويحفظ البيانات محلياً، مع ذكاء اصطناعي لاستخراج الطلبات من سكرين شوت أو من محادثة واتساب كاملة، وتصدير جمهور Meta المخصص.

بُني بـ **React + Vite + Tailwind CSS**، ويُنشر على **Netlify** مع دالة خادمية (Serverless Function) تحمي مفتاح Anthropic API.

---

## ✨ الميزات

- 📊 لوحة تحكم بالعمولات والإيرادات والتذكيرات
- 📸 استخراج بيانات الطلب من صورة محادثة (ذكاء اصطناعي)
- 🤖 استيراد محادثة واتساب كاملة (ملف `.txt`) واستخراج كل الزبائن دفعة واحدة
- 🔔 تذكيرات تلقائية بانتهاء السنسر مع رسالة واتساب جاهزة
- 💬 توليد رسالة للزبون ورسالة للمندوب (تُرسل للمندوب على رقم ثابت)
- 💰 احتساب عمولة المساعِدة (٢٠٠٠ جديد / ٣٠٠٠ تجديد)
- 👥 إدارة كاملة للزبائن (تعديل/حذف)
- 📤 تصدير CSV لجمهور Meta المخصص + تقرير مبيعات كامل
- 💾 نسخ احتياطي (تصدير/استيراد JSON)

أنواع السنسرات: **Libre 2** (٩٥ ألف) • **Libre 3 Plus** (١٢٥ ألف) • **Sibionics GS1** (٨٠ ألف) • **Sibionics أندرويد** (٦٥ ألف).

## 🚀 النشر على GitHub ثم Netlify (خطوة بخطوة)

### المتطلب الوحيد: مفتاح Anthropic API
احصل عليه من https://console.anthropic.com → API Keys. (يبدأ بـ `sk-ant-...`)

### الخطوة ١: رفع المشروع على GitHub
1. أنشئ حساباً على https://github.com إن لم يكن لديك.
2. أنشئ مستودعاً جديداً (New repository) باسم مثل `lcare-sales`.
3. ارفع كل ملفات هذا المجلد (اسحبها في صفحة المستودع → **uploading an existing file** → Commit).

> لا ترفع مجلد `node_modules` ولا ملف `.env`. ملف `.gitignore` يتجاهلهما تلقائياً.

### الخطوة ٢: الربط مع Netlify
1. أنشئ حساباً على https://www.netlify.com (سجّل دخول بحساب GitHub).
2. اضغط **Add new site → Import an existing project** → اختر **GitHub** → اختر مستودع `lcare-sales`.
3. Netlify يقرأ `netlify.toml` تلقائياً (Build: `npm run build`، Publish: `dist`). اتركها كما هي.
4. **مهم جداً — قبل Deploy:** افتح **Add environment variables** وأضِف:
   - Key: `ANTHROPIC_API_KEY`
   - Value: مفتاحك (`sk-ant-...`)
5. اضغط **Deploy**، وبعد دقيقة يعطيك رابطاً مثل `https://lcare-sales.netlify.app`.

## 🛠️ حل المشكلة: "الاستخراج لا يعمل على الموقع"

إذا فتح الموقع لكن استخراج المحادثة قال **"لم يتم العثور على طلبات"** بينما يعمل داخل كلود — فالسبب أن الدالة الخادمية لم تنجح. السبب الأشهر: **المفتاح غير مضبوط**. تحقّق هكذا:

### ١) اختبر الدالة مباشرة
افتح في المتصفح (بدّل الاسم باسم موقعك):
```
https://اسم-موقعك.netlify.app/api/extract
```
- ظهر `"apiKeyConfigured": true` → المفتاح مضبوط ✓ (انتقل للأسفل)
- ظهر `"apiKeyConfigured": false` → المفتاح ناقص (الخطوة ٢)
- ظهر خطأ 404 → الدالة غير مرفوعة (تأكد أن `netlify/functions/extract.js` و `netlify.toml` مرفوعان)

### ٢) أضِف المفتاح
Netlify → **Site configuration → Environment variables → Add a variable**:
- Key: `ANTHROPIC_API_KEY`
- Value: `sk-ant-...`

### ٣) أعد النشر (ضروري!)
المتغيرات الجديدة لا تُطبّق إلا بنشر جديد:
**Deploys → Trigger deploy → Deploy site** → انتظر الاكتمال → جرّب مرة أخرى.

> التطبيق الآن يعرض **سبب الخطأ الحقيقي** (مثلاً: المفتاح ناقص / غير صحيح / الخدمة مشغولة) بدل رسالة عامة.

## ☁️ تفعيل المزامنة السحابية بين الأجهزة (Supabase)

افتراضياً يحفظ التطبيق البيانات محلياً على كل جهاز. لتفعيل **المزامنة التلقائية بين جهازك وجهاز مريم** (وحماية البيانات من الضياع)، فعّل التخزين السحابي المجاني عبر Supabase:

### الخطوة ١: أنشئ مشروع Supabase
1. ادخل https://supabase.com → سجّل دخول (مجاني، بحساب GitHub)
2. اضغط **New project** → اختر اسماً وكلمة مرور لقاعدة البيانات → **Create**
3. انتظر دقيقة حتى يجهز المشروع

### الخطوة ٢: أنشئ الجدول
1. من القائمة اليسرى: **SQL Editor** → **New query**
2. افتح ملف **`supabase-setup.sql`** (المرفق في المشروع) → انسخ محتواه كاملاً → الصقه
3. اضغط **Run** (سينشئ الجدول ويفعّل المزامنة)

### الخطوة ٣: انسخ رابط المشروع والمفتاح
1. من القائمة اليسرى: **Settings** (الترس) → **API**
2. انسخ قيمتين:
   - **Project URL** (مثل `https://xxxxx.supabase.co`)
   - **anon / publishable key** (مفتاح عام طويل — آمن للمتصفح)

### الخطوة ٤: أضِف القيمتين في Netlify
1. Netlify → مشروعك → **Site configuration** → **Environment variables**
2. أضِف متغيرين:
   - `VITE_SUPABASE_URL` = رابط المشروع
   - `VITE_SUPABASE_ANON_KEY` = المفتاح العام
3. **أعد النشر:** Deploys → **Trigger deploy** → **Clear cache and deploy site**

تم! ✓ الآن أي تعديل على أي جهاز يظهر فوراً على الجهاز الآخر. سترى شارة **"مُزامَن ☁️"** بجانب اسم التطبيق بدل "محلي".

> **مهم — نقل بياناتك الحالية للسحابة:** إذا كان عندك بيانات محفوظة محلياً قبل تفعيل السحابة، افتح التطبيق على نفس الجهاز بعد التفعيل، وأضِف أي طلب جديد (أو استورد محادثة) — سيُرفع كل شيء تلقائياً للسحابة. أو استخدم "تصدير → نسخة احتياطية" قبل التفعيل، ثم "استيراد" بعده للأمان.

## 🔐 أين تُحفظ البيانات؟

- **بدون إعداد Supabase:** تُحفظ محلياً في متصفح كل جهاز (localStorage). خاصة بالجهاز، وقد تُحذف إذا مسحت بيانات المتصفح — لذا خذ نسخة احتياطية بانتظام.
- **مع إعداد Supabase (موصى به):** تُحفظ في السحابة وتتزامن بين كل الأجهزة تلقائياً، مع نسخة محلية احتياطية على كل جهاز. لا تُفقد عند مسح الكاش أو تغيير الجهاز.
- **خذ نسخة احتياطية دورية** من تبويب "تصدير" → "تنزيل نسخة احتياطية كاملة" في الحالتين.
- **مفتاح الذكاء الاصطناعي:** يُحفظ في Netlify فقط (خادمياً)، ولا يظهر في المتصفح أبداً.

## ❓ أسئلة أخرى

**هل بيانات هاتفي تنتقل لهاتف مريم تلقائياً؟**
لا — التخزين محلي لكل جهاز. للمشاركة: صدّر نسخة احتياطية واستوردها في الجهاز الآخر. (المزامنة التلقائية تحتاج قاعدة بيانات سحابية، يمكن إضافتها لاحقاً.)

**كيف أغيّر رقم المندوب؟**
عدّل `DELIVERY_PHONE` في `src/App.jsx` ثم ارفع التعديل (Netlify يعيد النشر تلقائياً).
